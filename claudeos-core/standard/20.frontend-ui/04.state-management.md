# 상태 관리

## 현재 상태 관리 전략

이 프로젝트는 로컬 `useState` 만 사용합니다.
전역 상태 라이브러리(Zustand, Redux, Jotai, Context API)는 도입되지 않았습니다.

| 상태 종류 | 현재 방법 | 권장 개선 |
|----------|-----------|-----------|
| 서버 데이터 | `useState` + `useEffect` | TanStack Query 도입 권장 |
| 전역 UI 상태 | 없음 (미사용) | 필요 시 Zustand |
| URL 상태 | 없음 (미사용) | `useSearchParams` 활용 권장 |
| 폼 상태 | `useState` | React Hook Form 권장 |

## 로컬 상태 패턴

```tsx
// ✅ 타입 안전한 상태 정의
const [stats, setStats] = useState<DashboardStats | null>(null)
const [loading, setLoading] = useState(true)
const [activeTab, setActiveTab] = useState<string>('inks')

// ✅ 관련된 값은 객체로 묶기 (Lab 입력값 예시)
const [labInput, setLabInput] = useState({ L: 50, a: 0, b: 0 })
// setter 사용 시
setLabInput(prev => ({ ...prev, L: 60 }))
```

```tsx
// ❌ 타입 없는 any 사용 금지
const [formData, setFormData] = useState<any>({})

// ❌ 관련 상태를 불필요하게 분리 금지
const [labL, setLabL] = useState(50)
const [labA, setLabA] = useState(0)
const [labB, setLabB] = useState(0)  // L, a, b는 하나의 객체로 관리
```

## 폼 상태

폼의 각 엔티티 타입에 맞는 인터페이스를 정의하고 초기값을 명시합니다.

```tsx
// ✅ 타입 안전한 폼 상태
interface InkFormData {
  name: string
  category: string
  opacity: number
}

const INITIAL_FORM: InkFormData = { name: '', category: 'process', opacity: 0 }
const [formData, setFormData] = useState<InkFormData>(INITIAL_FORM)

const handleChange = (field: keyof InkFormData, value: string | number) => {
  setFormData(prev => ({ ...prev, [field]: value }))
}
```

```tsx
// ❌ 폼 상태 타입 미지정 금지
const [form, setForm] = useState<any>({})
```

master 도메인의 탭 기반 멀티 엔티티 CRUD 패턴:

```tsx
// ✅ 탭 타입을 union type으로 정의
type TabKey = 'inks' | 'plates' | 'pads' | 'base_colors' | 'white_refs' | 'recipes'
const [activeTab, setActiveTab] = useState<TabKey>('inks')
```

## 로딩/에러 상태

```tsx
// ✅ 명확한 로딩/에러 상태 분리
const [loading, setLoading] = useState(true)
const [error, setError] = useState<string | null>(null)

useEffect(() => {
  api.get('/stats')
    .then(res => setStats(res.data))
    .catch(err => {
      console.error(err)
      setError('데이터를 불러오지 못했습니다.')
    })
    .finally(() => setLoading(false))
}, [])

if (loading) return <div>로딩 중...</div>
if (error) return <div style={{ color: 'red' }}>{error}</div>
```

```tsx
// ❌ 에러를 console.error만으로 처리 금지 — 사용자에게 표시 필요
.catch(err => console.error(err))  // 사용자는 에러를 알 수 없음
```

## URL 상태 (미래 개선)

현재 URL 상태를 활용하지 않아 북마크/공유가 불가합니다.
향후 `useSearchParams` 로 개선을 권장합니다.

```tsx
// 권장 개선 방향 (미래 구현)
import { useSearchParams, useRouter } from 'next/navigation'

export default function ComplementaryPage() {
  const searchParams = useSearchParams()
  const router = useRouter()
  const L = Number(searchParams.get('L') ?? 50)
  // URL에 Lab 값 저장 → 공유/북마크 가능
}
```

## 핵심 규칙 요약

| 규칙 | 내용 |
|------|------|
| 상태 라이브러리 | 현재 미사용 — `useState` 만 사용 |
| 폼 상태 타입 | 엔티티별 `interface` 정의 필수, `any` 금지 |
| 관련 상태 | 하나의 객체 상태로 묶기 |
| 로딩 상태 | `useState<boolean>` |
| 에러 상태 | `useState<string \| null>` + UI에 표시 |
| URL 상태 | 현재 미사용 (`useSearchParams` 활용 권장) |
