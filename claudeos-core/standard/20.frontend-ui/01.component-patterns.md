# 컴포넌트 패턴

## 기본 컴포넌트 구조

모든 컴포넌트는 함수형 컴포넌트로 작성하며, 파일 상단에 Props 인터페이스를 정의합니다.

```tsx
// ✅ 표준 컴포넌트 구조
interface Props {
  value: number
  label?: string
}

export default function OpacityGauge({ value, label }: Props) {
  return (
    <div style={{ position: 'relative' }}>
      {/* SVG 또는 JSX */}
    </div>
  )
}
```

```tsx
// ❌ 금지 패턴
// class 컴포넌트
class OpacityGauge extends React.Component { }

// forwardRef, memo 불필요하게 사용
const OpacityGauge = React.memo(React.forwardRef(...))

// Props 타입 미정의
export default function OpacityGauge({ value }) { }
```

## 공유 컴포넌트 목록

| 컴포넌트 | import 경로 | 용도 |
|----------|-------------|------|
| `OpacityGauge` | `from '@/components/OpacityGauge'` | SVG 반원 게이지 — 불투명도/은폐력 점수 시각화 |
| `InkDonutChart` | `from '@/components/InkDonutChart'` | SVG 도넛 차트 — 잉크 카테고리 분포 |
| `TransparencySimulator` | `from '@/components/TransparencySimulator'` | 잉크 투명도 시뮬레이터 |

## Props 패턴

```tsx
// ✅ 선택적 Props에 기본값 지정
interface Props {
  value: number        // 필수
  max?: number         // 선택적
  label?: string       // 선택적
}

export default function OpacityGauge({ value, max = 100, label = '' }: Props) { ... }
```

```tsx
// ❌ any 타입 사용 금지
interface Props {
  data: any            // 금지
  items: any[]         // 금지
}
```

## 컴포넌트 내 서브 컴포넌트

페이지 함수 내부에 서브 컴포넌트를 인라인으로 정의하지 마세요.
매 렌더마다 새 참조가 생성되어 성능 문제가 발생합니다.

```tsx
// ✅ 모듈 스코프에 서브 컴포넌트 정의
interface LabBoxProps {
  L: number; a: number; b: number
}
function LabBox({ L, a, b }: LabBoxProps) {
  return <div>L: {L}, a: {a}, b: {b}</div>
}

export default function MasterPage() {
  return <LabBox L={50} a={20} b={10} />
}
```

```tsx
// ❌ 페이지 함수 내부에 컴포넌트 정의 금지
export default function MasterPage() {
  // 렌더마다 새 함수 참조 생성 — 성능 저하
  const LabBox = ({ L, a, b }: LabBoxProps) => <div>...</div>
  return <LabBox L={50} a={20} b={10} />
}
```

## 상수 정의

컴포넌트에서 사용하는 상수는 파일 모듈 스코프(컴포넌트 함수 외부)에 정의합니다.
여러 파일에서 공유하는 상수는 중복 정의하지 말고 단일 소스로 관리하세요.

```tsx
// ✅ 모듈 스코프에 상수 정의
const CATEGORY_LABELS: Record<string, string> = {
  process: '프로세스',
  special: '특수',
  metallic: '메탈릭',
}

export default function InkDonutChart({ data }: Props) { ... }
```

```tsx
// ❌ 동일 상수를 여러 파일에서 중복 정의 금지
// master/page.tsx 와 InkDonutChart.tsx 양쪽에 CATEGORY_LABELS 정의 — 불일치 위험
```

## ΔE 색상 임계값 상수

프로젝트 표준 ΔE 색상 임계값 (대시보드·보색 도메인 공통):

```ts
// ✅ 프로젝트 표준 ΔE 임계값
const DELTA_E_COLOR = (value: number): string => {
  if (value < 1) return '#10b981'   // 초록 — 매우 양호
  if (value < 3) return '#3b82f6'   // 파랑 — 양호
  if (value < 6) return '#f59e0b'   // 노랑 — 주의
  return '#ef4444'                   // 빨강 — 불량
}
```

## 핵심 규칙 요약

| 규칙 | 내용 |
|------|------|
| 컴포넌트 유형 | 함수형만 사용 (class, forwardRef, memo 불사용) |
| Props 타입 | `interface Props { }` 필수 |
| export | `default export` |
| 서브 컴포넌트 | 모듈 스코프에 정의 (함수 내부 정의 금지) |
| 상수 | 모듈 스코프에 UPPER_SNAKE_CASE로 정의 |
| any 타입 | 사용 금지 |
| import 경로 | `@/components/ComponentName` |
